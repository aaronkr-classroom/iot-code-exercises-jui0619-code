# chp1 연습문제

1.Q1
  - (a) 디바이스의 센서와 액추에이터의 역할을 비교하여 설명하세요.
  > 센서는 외부 환경의 온도, 습도, 빛과 같은 변화를 감지하여 전기 신호로 변환하는 입력 장치이다. 하지만 액추에이처는 전기 신호를 받아서 동작으로 변환하는 출력장치다. 따라서 센서는 주변의 환경을 데이터로 바꾸는 입력 장치이고, 액추에이터는 데이트를 주변의 환경으로 변환하는 출력 장치이다.
 
  - (b) 디바이스와 대시보드에서 동일한 프로그래밍 언어를 사용하지 않는 이유를 설명하세요.
  > 디바이스는 메모리와 처리 능력이 많이 제한적이라 가벼운 언어가 필요하다. 하지만 대시보드는 웹 브라우저에서 실행이 되기 때문에 자바 스크립트나 html언어를 사용하는 것이 좋다. 따라서 각자 최적화된 언어를 사용하는 것이다.
  
  - (c) 브로커와 같은 정보 전달 전담 서버의 이점을 설명하세요.
  > 브로커를 사용하면 발행자와 구독자가 서로 직접 연결될 필요가 없어진다. 따라서 디바이스는 브로커에게만 메세지를 보내면 되기에 전달 구도가 단순해진다. 새 디바이스를 추가하거나 없애도 다른 장치에 영향이 없는 것도 브로커의 이점 중 하나다. 게다가 브로커가 메세지를 버퍼링하므로 받는 쪽이 잠시 오프라인이어도 데이터 손실을 줄일 수 있다.

2. Q2
  - (a) Mosquitto와 Node-RED를 별도 서버에 설치할 경우의 이점을 설명하세요.
  - (b) Mosquitto와 Node-RED를 운영하기 위한 적절한 컴퓨팅 환경을 기술하세요.
  - (C) Mosquitto와 Node-RED에 대해 설명, 실행에 적합한 컴퓨팅 환경을 제시하세요.
  > Mosquitto는 MQTT 브로커로, IoT 디바이스 사이의 메세지를 중간에서 연결해주는 역할을 한다. 메모리 사용이 적어서 소형 보드에서도 충분히 실행이 가능하다. Node-RED는 브라우저 기반의 프로그래밍 도구로, 드래그랑 드롭 방식을 사용하여 IoT 대시보드를 구성한다. 웹 서버를 갖고 있으므로 일반 컴퓨터, 클라우드 서버에서 실행하는 것이 좋다.

3. 3개 이상의 개발 보드를 찾아서 특징과 장단점을 비교하세요.

4. 스마트 홈 구축을 위한 아이디어를 다음 조건을 준수하여 정리해보세요. 
  - 전원 제어, 보안, 환경 제어 포함
  - 사용할 센서 목록 작성
  - 대시보드 구성 항목 정리
  - 시스템 통제 시나리오 작성
